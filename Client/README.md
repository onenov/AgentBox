# AgentBox Client

`Client/` 是 AgentBox 的前端与桌面客户端工程，负责提供 Web 控制台、Tauri 桌面壳、主题配置、运行环境检测视图，以及 OpenClaw、Hermes、CC-Connect 的管理界面。

## 技术栈

- React 19
- TypeScript
- Vite 8
- Tailwind CSS 4
- HeroUI / HeroUI Pro
- React Router
- Zustand
- Tauri 2
- Monaco Editor
- xterm.js
- Recharts

## 本地开发

安装依赖：

```bash
pnpm install
```

创建本地环境文件：

```text
FRONTEND_DEV_PORT=5173
```

可以直接写入：

```bash
printf 'FRONTEND_DEV_PORT=5173\n' > .env
```

启动前端开发服务：

```bash
pnpm dev
```

默认地址：

```text
http://127.0.0.1:5173
```

`pnpm dev` 会读取 `Client/.env` 中的 `FRONTEND_DEV_PORT`，并以 strict port 模式启动 Vite。端口被占用时会直接失败，方便避免前后端地址混乱。

开发时后端由 `Server/` 单独运行：

```bash
cd ../Server
go run github.com/air-verse/air@latest
```

前端默认调用：

```text
http://127.0.0.1:8787
```

这个地址来自 `public/config.js` 的 `API_URL`。

## 脚本说明

| 命令 | 说明 |
| --- | --- |
| `pnpm dev` | 启动 Vite 开发服务 |
| `pnpm lint` | 运行 ESLint |
| `pnpm preview` | 预览 Vite 构建产物 |
| `pnpm tauri` | 透传 Tauri CLI |
| `pnpm tauri:dev` | 清理 Tauri 缓存、同步 dev URL、启动桌面开发模式 |
| `pnpm tauri:build` | 同步版本并构建桌面端 |
| `pnpm clean:tauri-cache` | 清理 Tauri 开发缓存 |

日常开发只需要 `pnpm dev`。除非明确要做发布验证，不需要执行 `pnpm build` 或 `pnpm tauri:build`。

## 运行时配置

前端运行时配置在：

```text
public/config.js
```

主要字段：

| 字段 | 说明 |
| --- | --- |
| `APP_VERSION` | 应用版本 |
| `APP_NAME` | 应用名称 |
| `APP_LOGO` | 品牌图标 |
| `HOME_ROUTE` | 首页路由 |
| `API_URL` | 后端 API 地址 |
| `MODEL_CATALOG_URL` | 模型目录地址 |
| `MODEL_INITIALIZATION_URL` | 模型初始化配置地址 |
| `MODEL_ICON_BASE_URL` | 模型图标基础地址 |
| `NOTICE_URL` | 公告地址 |
| `ABOUT_URL` | 关于页数据地址 |
| `CONNECT_URL` | 社群数据地址 |
| `PROMOTION_URL` | 推荐内容地址 |
| `UPDATE_URL` | Tauri 更新清单地址 |
| `THEME_STYLE` | 默认主题风格 |
| `THEME_COLOR` | 默认主题颜色 |
| `THEME_FONT` | 默认字体 |

前端启动后会通过 `src/stores/config.ts` 读取 `window.APP_CONFIG`。用户在界面里修改 API 地址时，会写入 localStorage/sessionStorage；Tauri 环境下还会同步到桌面偏好存储。

## 路由约定

项目使用 `vite-plugin-pages` 生成文件路由：

```ts
Pages({
  dirs: [
    { dir: 'src/pages/public', baseRoute: '' },
    { dir: 'src/pages/dashboard', baseRoute: 'dashboard' },
  ],
})
```

因此：

- `src/pages/public/login/index.tsx` 对应 `/login`
- `src/pages/public/openclaw-chat/index.tsx` 对应 `/openclaw-chat`
- `src/pages/dashboard/openclaw/index.tsx` 对应 `/dashboard/openclaw`
- `src/pages/dashboard/system/index.tsx` 对应 `/dashboard/system`

新增页面时优先放到对应业务目录。Dashboard 导航配置在：

```text
src/layouts/Dashboard/nav.ts
```

## API 分层

API 请求入口：

```text
src/api/client.ts
```

它负责：

- 读取当前 API Base URL
- 拼接 query 参数
- 自动附带 `Authorization: Bearer <token>`
- 处理 401 并跳转登录页
- 解析 JSON 错误信息

业务 API 按功能拆分：

| 文件 | 说明 |
| --- | --- |
| `auth.ts` | 登录、认证配置 |
| `environment.ts` | 主机环境检测 |
| `network.ts` | 网络检测 |
| `proxy.ts` | 后端代理设置 |
| `logging.ts` | 后端日志 |
| `maintenance.ts` | 维护操作 |
| `plugins.ts` | AgentBox 插件 |
| `openclaw.ts` | OpenClaw 管理接口 |
| `openclaw-channels.ts` | OpenClaw 渠道接口 |
| `openclaw-ws.ts` | OpenClaw WebSocket 相关能力 |
| `hermes.ts` | Hermes 管理接口 |
| `cc-connect.ts` | CC-Connect 管理接口 |

新增接口时优先复用 `apiRequest`，不要在页面里直接拼 `fetch`。

## 状态管理

Zustand store 位于：

```text
src/stores/
```

常见职责：

- `config.ts`：运行时配置、API 地址、主题默认值
- `theme.ts`：主题状态
- `environment.ts`：系统环境检测状态
- `openclaw-environment.ts`：OpenClaw 环境状态
- `hermes-environment.ts`：Hermes 环境状态
- `cc-connect-environment.ts`：CC-Connect 环境状态
- `notice.ts`：公告状态
- `update.ts`：桌面更新状态

页面内短生命周期状态可以留在组件里；跨页面共享、需要持久化或被多个组件消费的状态再提升到 store。

## 认证与登录

前端 token 存储 key：

```text
agent-box-auth-token
```

支持两种持久化方式：

- `persistent`：写入 localStorage，并同步到 Tauri 桌面偏好
- `session`：写入 sessionStorage，不跨会话持久化

API 请求默认使用 Bearer Token。未认证或 token 失效时，`apiRequest` 会跳转到 `/login`，并保留原始 redirect 地址。

桌面端工程在：


常用文件：

| 文件 | 说明 |

```bash
pnpm tauri:dev
```

这个命令会先执行：

- `scripts/clean-tauri-cache.mjs`
- `scripts/sync-tauri-dev-url.mjs`

然后启动 `tauri dev`。

## 常见问题

### `pnpm dev` 读取 `.env` 失败

`scripts/dev.mjs` 会读取 `Client/.env`。如果本地没有该文件，创建一个最小配置：

```text
FRONTEND_DEV_PORT=5173
```

### 前端请求后端失败

检查：

- 后端是否运行在 `http://127.0.0.1:8787`
- `public/config.js` 的 `API_URL` 是否正确
- 浏览器 localStorage 是否覆盖了旧的 API 地址
- 后端是否需要登录 token

需要重置 API 地址时，可以在界面里重置，也可以清理 localStorage 的：

```text
agent-box-api-url
```

### HeroUI Pro 依赖安装失败

项目依赖 `@heroui-pro/react`。如果安装时遇到私有包认证问题，需要确认本地 npm/pnpm registry 认证配置已正确设置。不要把 npm token 写入仓库。

## 提交边界

不要提交：

- `node_modules/`
- `dist/`
- `.env`
- `src-tauri/target/`
- `src-tauri/binaries/` 下的真实 sidecar 二进制
- 本地 Tauri 签名密钥、证书或发布包

可以提交：

- `src-tauri/binaries/.gitkeep`
- `public/config.js`
- `src/` 源码
- `src-tauri/` 配置、Rust 源码、权限文件和图标
- `pnpm-lock.yaml`

提交前建议执行：

```bash
git status --short --ignored
```
